🧱 Organização do Código e da Estrutura
• Divida arquivos grandes em arquivos menores e funções longas em funções menores.
• Mantenha o código limpo, bem estruturado e legível.
• Evite arquivos com mais de 200–300 linhas.
• Crie componentes modulares e reutilizáveis.
• Separe responsabilidades com clareza.
• Organize os arquivos por função ou recurso.
• Use nomes claros, consistentes e fáceis de entender.
• Padronize nomes de arquivos, pastas e estruturas com base nas convenções do projeto.
• Evite scripts soltos em arquivos principais (ex: index.js, main.py).
• Use dados simulados (mocks) apenas em testes automatizados e locais.
• Nunca sobrescreva arquivos sensíveis como .env sem confirmação explícita.
• Sempre inclua os imports necessários, sem deixar dependências implícitas.
• Nomeie funções de evento com prefixo handle, ex: handleClick, handleKeyDown.
• Prefira usar const ao declarar funções, e defina tipos sempre que possível.

🧠 Consciência de Impacto Sistêmico
Sempre que alterar uma classe, siga estas diretrizes:

• Mapeie todo o fluxo relacionado à classe modificada.
• Verifique quais outras classes, módulos ou componentes dependem dela.
• Altere o mínimo necessário, mas revise o todo.
• Ao alterar o frontend, garanta que o backend esteja em conformidade.
• Ao alterar o backend, garanta que o contrato com o frontend continue válido.
• Verifique se os testes cobrem o novo cenário.
• Se necessário, atualize a documentação e as variáveis de ambiente.
• Nunca finalize uma alteração sem considerar os efeitos colaterais no sistema.
• Alinhe com o time se houver dúvidas sobre impacto cruzado.

🔁 Reutilização e Consistência
• Prefira soluções simples (KISS).
• Evite duplicação (DRY).
• Antes de criar algo novo, verifique se já existe.
• Ao corrigir bugs, siga os padrões e tecnologias já utilizados.
• Só introduza novos padrões ou tecnologias se necessário — e remova completamente o antigo.
• Ao implementar algo novo, proponha refatorações que melhorem a estrutura do projeto como um todo.
• Sempre procure e remova lixo deixado por mudanças anteriores.
• Não criamos soluções temporárias ou mocks para "quebrar o galho". Se algo estiver faltando, implementamos corretamente e depois seguimos.

🌍 Consciência de Ambiente
• Escreva código considerando os ambientes dev, test e prod.
• Trate erros com mensagens amigáveis.
• Implemente estados de carregamento e fallback para falhas de rede.
• Crie índices e otimize consultas para melhorar a performance.
• Use transações em operações críticas de banco de dados.
• Adicione métricas e logs estruturados sempre que possível.

🔐 Boas Práticas de Configuração e Segurança
• Siga o princípio da configuração via ambiente: nunca deixe valores sensíveis no código.
• Use arquivos .env apenas em desenvolvimento local e nunca os versione.
• Utilize Secrets Managers (ex: AWS Secrets Manager, Vault, Google Secret Manager) em produção.
• Configure ferramentas para detectar strings hardcoded (ex: linters, CI hooks).
• Nunca exponha tokens, chaves de API ou senhas em logs, prints ou commits.
• Padronize variáveis de ambiente com prefixos e nomes descritivos.
• Valide variáveis obrigatórias no boot da aplicação.
• Tenha uma camada de abstração centralizada para leitura de configurações.
• Documente todas as variáveis por ambiente em local único e acessível.

♿ Acessibilidade e UX
• Todo elemento interativo deve ter tabIndex, aria-label, eventos onClick e onKeyDown.
• Priorize acessibilidade sem comprometer a experiência do usuário.
• Use frameworks modernos (ex: TailwindCSS, Shadcn, Radix) com consciência de boas práticas de UI/UX.
• Utilize class: em vez de operadores ternários em className, sempre que possível, para facilitar leitura.

• Reduza para 1 ou 2 causas mais prováveis.
• Adicione logs para validar hipóteses.
• Use ferramentas como: getConsoleLogs, getConsoleErrors, getNetworkLogs, getNetworkErrors.
• Obtenha logs do servidor ou peça que eu envie.
• Faça uma análise profunda e abrangente.
• Sugira novos pontos de log se necessário.
• Após corrigir, peça autorização para remover logs temporários.
